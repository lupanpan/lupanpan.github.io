git的工作流程

  1. 下载代码

    • 分配git账号 或者自己生成公钥和私钥，然后把公钥提交给git管理者；
    • git clone 远程仓库的代码；
    • 在下载的代码基础之上做开发，然后提交；

  2. 新建项目

    • 在服务器新创建一个新仓库；
    • 在本地git init生成一个仓库，在这里进行开发，开发完成后提交代码；
    • 然后推送到服务器的空仓库中；

git中的三个重要区域

  1. 工作目录

    对项目的某个版本独立提取出来的内容，这些从Git仓库的压缩数据库中提取出来的文件，放在磁盘上供你使用或修改。

  2. 暂存区域

    是一个文件，保存了下次将提交的文件列表信息，一般在git仓库目录中，有时候被称为索引，不过一般说法还是叫暂存区域。

  3. 本地仓库

    是git用来保存项目的元数据和对象数据库的地方，这是git中最重要的部分，从其它计算机克隆仓库时，拷贝的就是这里的数据。

git文件的四种状态

  1. 未跟踪 untracked

    工作目录刚创建的文件；

  2. 已暂存 staged

    从工作目录提交到暂存区的文件；

  3. 已提交 commited

    从缓存区提交到本地仓库的文件；

  4. 已修改 modified

    暂存区修改过的文件；
